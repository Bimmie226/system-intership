# Tìm hiểu Repository và các loại Repository phổ biến
## I. Repository là gì? 
Repository (repo) là một không gian lưu trữ mã nguồn, nó có vai trò quan trọng trong quá trình phát triển phần mềm hiện đại và hỗ trợ kiểm soát phiên bản 

Bạn có thể tưởng tượng Repository như một thư mục dự án, nơi theo dõi các lịch sử thay đổi, các đóng góp và hoạt động phân nhánh. Thư mục này chứa toàn bộ các file code tạo thành ứng dụng hoàn chỉnh

Bên cạnh các tính năng chính như phiên bản hóa (versioning), lịch sử thông tin người tạo, thời gian mà các tệp được tạo ra, Repository còn có nhiều tính năng hữu ích khác như thêm, xóa hoặc sửa đổi file. Những tính năng này giúp quản lý source code dễ dàng, thuận lợi cho làm việc trong repo để phát triển sản phẩm

## II. Các loại Repository phổ biến
### 2.0 Local Repository
Local Repository là bản sao hoàn chỉnh của Repository được dùng để lưu trữ, quản lý và theo dõi các file source code. Khi khởi tạo 1 dự án Git hoặc clone 1 Repo từ xa, hệ thống sẽ tạo ra một Local Repository. Mọi thay đổi bạn thực hiện sẽ được ghi lại và lưu trữ cục bộ trong kho lưu trữ này

Tính năng cốt lõi của Local Repository là cho phép lập trình viên thực hiện các thao tác trong môi trường không cần internet

### 2.1 Remote Repository
Remote Repository là phiên bản chính thức của dự án, được lưu trữ trên 1 server từ xa và có thể truy cập được qua mạng. Các nền tảng để lưu trữ Remote Repository là `Github`, `Gitlab`, `Bitbucket`. Thể mạnh của Remote Repository là khả năng chia sẻ mã nguồn và cộng tác nhóm tuy nhiên các thao tác đều cần kết nối Internet

Để cập nhật và chia sẻ công việc, các lập trình viên thường đồng bộ hóa (push/pull) giữa Local Repository và Remote Repository. Quá trình này nhằm đảm bảo tính nhất quán của cơ sở mã giữa tất cả các thành viên trong nhóm, đồng thời lưu trữ lịch sử chỉnh sửa theo trình tự thời gian.
