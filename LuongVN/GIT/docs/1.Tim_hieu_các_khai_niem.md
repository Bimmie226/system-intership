# GIT Introduction 

## I. Version Control System (VCS) 
Version Control System (VCS) là hệ thống kiểm soát các phiên bản phân tán mã nguồn mở. Các VCS sẽ lưu trữ tất cả các file của dự án và lưu lại toàn bộ lịch sử thay đổi của file. Mỗi sự thay đổi được lưu lại là 1 version

VCS là hệ thống giúp lập trình viên có thể lưu trữ nhiều phiên bản khác nhau của một mã nguồn được `clone` từ một `repository`, mỗi thay đổi vào mã nguồn trên local có thể `commit` rồi đẩy lên server nơi đặt `repository` chính

## II. Git là gì?
Git là một Version Control System (VCS) được tạo ra bởi Linus Torvalds vào năm 2005, Git cho phép nhiều lập trình viên làm việc song song trên cùng một dự án mà không ghi đè lên nhau:
- Theo dõi thay đổi của code
- Theo dõi ai là người đã thay đổi code

## III. Github là gì?
Github là một hệ thống quản lý dự án, lưu trữ source code, theo dõi và cộng tác trong quá trình phát triển phần mềm

Các lập trình viên có thể clone lại mã nguồn từ một repository và Github chính là một dịch vụ máy chủ repository công cộng, mỗi người có thể tạo tài khoản trên đó để tạo ra các repo của riêng mình để có thể làm việc

Github là một dịch vụ nổi tiếng cung cấp kho lưu trữ mã nguồn `Git` cho các dự án phần mềm. Github có đầy đủ những tính năng của Git, ngoài ra nó còn bổ sung những tính năng về social để các developer tương tác với nhau.

### 3.0 Một vài khái niệm của Git
- `git`: là prefix của các lệnh được sử dụng dưới CLI
- `branch`: nhánh - thể hiện sự phân chia các version 
- `folk`: Folk là thao tác thực hiện sao chép repository của chủ sở hữu khác về git account của mình. sử dụng như 1 repository do mình tạo ra.
- `tag`: sử dụng để đánh dấu một commit
- `remote`: sử dụng để điều khiển các nhánh từ một repository trên git server
- `diff`: So sánh sự sai khác giữa phiên bản hiện tại với phiên bản muốn so sánh, nó sẽ thể hiện các sự khác nhau
- `.gitignore`: file mặc định của git sử dụng để loại bỏ (ignore) các thư mục, file mà mình không muốn push lên git server

## IV. Bitbucket là gì?
Bitbucket là một hệ thống quản lý kho lưu trữ `Git` (Git repository) được thiết kế đặc thù cho các nhóm lập trình. Hệ thống này hoạt động như một trung tâm tập trung, nơi xử lý mọi Git repository và quản lý source code trong suốt quá trình phát triển phần mềm 

Bitbucket thuộc sở hữu của `Atlassian`, nhà cung cấp các công cụ nổi tiếng như `Jira` và `Confluence`. 

### 4.0 Pipeline Bitbucket là gì?
Pipeline Bitbucket là một công cụ được tích hợp sẵn vào Bitbucket giúp hỗ trợ tự động hóa các quy trình làm việc (CI/CD). Với công cụ này bạn có thể tự động hóa các giai đoạn xây dựng, kiểm tra và triển khai mã code 

Pipeline Bitbucket có thể:
- `Tự động xây dựng mã`: Mỗi lần có các cập nhật trong kho lưu trữ, công cụ sẽ tự động xây dựng mã, đảm bảo mã luôn được cập nhật và sẵn sàng triển khai mọi lúc
- `Kiểm tra tự động`: Công cụ này tự động thực hiện các bài kiểm tra để xác định và khắc kịp thời các lỗi trong mã, giúp cải thiện chất lượng của sản phẩm.
- `Triển khai tự động`: Với tính năng tự động của công cụ, mã của bạn sẽ được chuyển đến môi trường production nhanh chóng và chính xác

## V. Gitlab là gì?
Gitlab được thiết kế để quản lý vòng đời phát triển phần mềm. Nền tảng cung cập một môi trường tích hợp mạnh mẽ, hỗ trợ các nhóm phát triển phần mềm trong mọi giai đoạn, từ lập kế hoạch, quản lý mã nguồn, kiểm tra mã, quản lý vấn đề, triển khai tự động cho đến theo dõi quy trình phát triển.

Các tính năng chính của Gitlab bao gồm:
- `Quản lý mã nguồn Git`: Lưu trữ, theo dõi các thay đổi trên code
- `Theo dõi vấn đề (Issue Tracking)`: Báo cáo, phân công và theo dõi lỗi, bug trong dự án
- `CI/CD`: Tự động hóa các tác vụ xây dựng, kiểm thử và triển khai phần mềm.
- `Yêu cầu hợp nhất (Merge Request)`: Kiểm tra và tích hợp các thay đổi code từ các thành viên khác.

### 5.0 Các phiên bản của Gitlab
Gitlab hỗ trợ 3 phiên bản sau:
- `Gitlab Community Edition (CE)`: Là bản open source miễn phí cho cộng đồng, phát hành qua hệ thống `Git`, thường xuyên được cập nhật tại các nhánh phát triển và nhánh ổn định bởi đội ngũ phát triển chính thức.
- `Gitlab Enterprise Edition (EE)`: Được thiết kế cho doanh nghiệp với nhiều chức năng nâng cao, phân phối trực tiếp từ website Gitlab, kèm theo dịch vụ hỗ trợ kỹ thuật trong quá trình phát triển và triển khai thực tế
- `Gitlab Continuous Integration (CI)`: Là nền tảng tự động hóa quy trình tích hợp và kiểm thử liên tục, do nhóm phát triển GitLab xây dựng, giúp tăng tốc độ và nâng cao chất lượng triển khai phần mềm.

### 5.1 Protected branches
Với đặc điểm này, Gitlab cho phép đọc và ghi vào repo và các branches. Protected branches cấp quyền cho người được phép commit và push code. Nó có 3 đặc điểm sau:
- Ngăn chặn push từ mọi người trừ các user và master 
- Ngăn được việc push code lên branch từ những người không có quyền truy cập
- Chặn được những người thực hiện xóa branch

Master branch sẽ mặc định là protected branch còn user sẽ được cấp ít quyền hơn để đảm bảo tính bảo mật được an toàn.
