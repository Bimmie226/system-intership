# Triển khai Webserver trên cụm K8s 

## I. Nội dung 

- Tạo webserver chạy trên docker với k8s 
- Thử nghiệm các tính năng: 
  - scale, load balancing 
  - Update images cho deployments
  - roll back 

## II. Thực hiện 

### 2.0 Tạo file deploy apache 

```bash 
mkdir ~/project
vim ~/project/apache-deployment-service.yaml
```

Nội dung: 

```yaml
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: apache2
spec:
  replicas: 1

  selector:
    matchLabels:
      app: apache2

  template:
    metadata:
      labels:
        app: apache2

    spec:
      containers:
      - name: apache2
        image: cosy294/swarm:1.0

        ports:
        - containerPort: 80

---
apiVersion: v1
kind: Service
metadata:
  name: apache2

spec:
  type: NodePort

  selector:
    app: apache2

  ports:
  - port: 5555
    targetPort: 80
    nodePort: 30080
```

#### 2.2.2 Autoscale 

Autoscale sẽ tự động tạo ra các endpoint trên cụm cluster khi mà các Endpoint hiện tại không đáp ứng được khả năng xử lý. Nhằm giải quyết bài toán đáp ứng yêu cầu của người dùng trong từng thời điểm khác nhau.

```bash
devops@k8s-master-01:~/project$ kubectl autoscale deployments apache2 --max=10 --cpu-percent=80
horizontalpodautoscaler.autoscaling/apache2 autoscaled
```

### 2.3 Thử nghiệm tính năng update images, roll back

#### 2.3.1 Update với file cấu hình yaml 

Ta có file cấu hình mới: `apache-deployment-service-v2.yaml`

Với nội dung như sau: 

```bash
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: apache2
spec:
  replicas: 1

  selector:
    matchLabels:
      app: apache2

  template:
    metadata:
      labels:
        app: apache2

    spec:
      containers:
      - name: apache2
        image: nginx

        ports:
        - containerPort: 80

---
apiVersion: v1
kind: Service
metadata:
  name: apache2

spec:
  type: NodePort

  selector:
    app: apache2

  ports:
  - port: 5555
    targetPort: 80
    nodePort: 30080
```

- Image chuyển từ: `cosy294/swarm:1.0` lên `cosy294/swarm:2.0`


Sau đó apply lại: 

```bash
kubectl apply -f apache-deployment-service-v2.yaml
```

Kiểm tra rollout: 

```bash
devops@k8s-master-01:~/project$ kubectl rollout status deployment/apache2
deployment "apache2" successfully rolled out
```

Sau khi update toàn bộ Pod sẽ được update lên image mới. Ở đây sử dụng webserver là nginx chứ không phải apache như lúc đầu

#### 2.3.3 Rollback 

Rollback deployment apache2 quay về phiên bản trước đây, tức là ở đây deployment apache2 sẽ từ `cosy294/swarm:1.0` về `nginx`:

```bash
devops@k8s-master-01:~/project$ kubectl rollout undo deployments apache2
deployment.apps/apache2 rolled back
```
