# Upgrade Kubernetes Version

Kubernetes upgrade là quá trình cập nhật hệ thống Kubernetes lên phiên bản mới hơn. Đây là một tác vụ phức tạp, đòi hỏi phải được lập kế hoạch và thực hiện một cách cẩn thận. Quá trình nâng cấp có thể bao gồm việc cập nhật các thành phần như Kubernetes API Server, Controller Manager, Scheduler, Kubelet, Kube-Proxy, ... 

Điều quan trọng là việc nâng cấp Kubernetes không chỉ đơn thuần là chuyển sang một phiên bản mới hơn. Mục tiêu của quá trình này là đảm bảo toàn bộ hệ thống vẫn tiếp tục hoạt động ổn định sau khi nâng cấp, không gây gián đoạn với các ứng dụng đang chạy. Do đó, cần cân bằng giữa việc tận dụng các tính năng mới, các bản vá bảo mật với yêu cầu duy trì tính ổn định và độ tin cậy của hệ thống 

# Upgrading kubeadm Cluster 

Nâng cấp 1 cụm K8s được tạo bằng kubeadm. **Lưu ý:** K8s không hỗ trợ bỏ qua các phiên bản Minor trong quá trình nâng cấp. 

K8s khuyến nghị nên nâng cấp lên phiên bản Patch mới nhất càng sớm càng tốt, đồng thời đẩm bảo cụm của bạn luôn chạy một phiên bản Minor còn được hỗ trợ.

## I. Upgrade Workflow 

Ở mức tổng quan, quy trình nâng cấp Kubernetes bằng kubeadm bao gồm các bước sau: 

1. Nâng cấp Primary Control Plane Node 
2. Nâng cấp các Control Plane Node còn lại 
3. Nâng cấp các Worker Node 

## II. Những lưu ý khi nâng cấp etcd 

Do kube-apiserver được triển khai dưới dạng Static Pod, nên nó sẽ luôn chạy, kể cả khi đã drain Control Plane Node 

Nếu quá trình kubeadm upgrade bao gồm cả việc nâng cấp etcd, thì trong lúc static Pod của etcd được khởi động lại, mọi request đang được gửi đến kube-apiserver sẽ bị tạm dừng cho đến khi etcd hoạt động trở lại 

Để giảm thiểu điều này, K8s khuyến nghị chủ động dừng kube-apiserver 1 cách an toàn vài giây trước khi chạy lệnh `kubeadm upgrade apply` 

Thực hiện trên mỗi Control Plane Node như sau: 

```bash
killall -s SIGTERM kube-apiserver   # Thực hiện graceful shutdown cho kube-apiserver

sleep 20                            # Chờ khoảng 20 giây để các request hiện tại hoàn thành

kubeadm upgrade ...                 # Thực hiện lệnh nâng cấp kubeadm
```

## III. Change The Kubernetes Package Repository 

hướng dẫn cách kích hoạt (enable) kho lưu trữ gói (package repository) tương ứng với phiên bản Minor của Kubernetes mà bạn muốn nâng cấp.

**Lưu ý:** 

- Bước này chỉ cần thực hiện khi nâng cấp từ một Minor Version sang Minor Version khác 
- Ví dụ: 
  - v1.31 -> v1.32
  - v1.32 -> v1.33
- Nếu bạn chỉ nâng cấp Patch Version trong cùng 1 Minor Version (v1.32.5 -> v1.32.7) thì không cần làm theo hướng dẫn này 
- Tuy nhiên, nếu hệ thống của bạn vẫn đang sử dụng các kho lưu trữ cũ, bạn phải chuyển sang các kho lưu trữ mới do cộng đồng quản lý trước khi tiến hành nâng cấp.
- Các kho lưu trữ cũ: `apt.kubernetes.io` hay `yum.kubernetes.io` đã hcinsh thức ngừng phát triển từ ngày 13/09/2023
- K8s khuyến nghị và yêu cầu sử dụng các kho lưu trữ mới tại `pkg.k8s.io` để cài đặt các phiên bản K8s được phát hành sau ngày 13/09/2023

### 3.0 Kiểm tra hệ thống đang sử dụng kho lưu trữ K8s nào 

Hiển thị nội dung của tệp cấu hình khai báo APT Repository của K8s: 

```bash
pager /etc/apt/sources.list.d/kubernetes.list
```

Nếu kết quả như sau: 

```bash
devops@k8s-master-01:~$ pager /etc/apt/sources.list.d/kubernetes.list
deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] https://pkgs.k8s.io/core:/stable:/v1.30/deb/ /
```

Thì có nghĩa là hệ thống của bạn đang sử dụng kho lưu trữ gói K8s mới

### 3.1 Chuyển sang một Kubernetes Package Repository khác 

Bước này phải được thực hiện khi nâng cấp từ một Minor Version sang Minor Version khác, nhằm giúp hệ thống có thể truy cập và cài đặt các gói phần mềm của Minor Version mới.

```bash
vim /etc/apt/sources.list.d/kubernetes.list
```

![alt text](../images/upgrade_01.png)

- Bạn hãy thay version mà bạn muốn upgrade lên (lưu ý: không được bỏ qua bất kì minor nào), ở đây tôi sẽ nâng cấp từ 1.30 lên 1.31
