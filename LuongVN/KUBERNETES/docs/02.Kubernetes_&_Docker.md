# Điểm khác biệt giữa Kubernetes và Docker là gì ?

Docker là 1 công nghệ container cho phép xây dựng, kiểm thử và triển khai ứng dụng nhanh hơn so với các phương pháp truyền thống. Docker đóng gói các phần mềm thành các đơn vị chuẩn hóa gọi là container, chứa mọi thứ cần thiết để phần mềm hoạt động

Kubernetes là 1 công cụ điều phối container, cho phép mở rộng các hệ thống container để quản lý, phối hợp và lập lịch chạy container ở quy mô lớn 

## I. Kubernetes và Docker được sử dụng trong những trường hợp nào ? 

Cả k8s và Docker đều là các công nghệ liên quan đến container 

Các ứng dụng hiện đại thường được xây dựng theo kiến trúc microservices - gồm nhiều thành phần độc lập, mỗi thành phần chạy một quy trình ứng dụng riêng dưới dạng một dịch vụ. Mỗi dịch vụ thực hiện một chức năng cụ thể và giao tiếp với các dịch vụ khác thông qua API 

### 1.0 Tạo container 

Docker cung cấp 1 bộ công cụ giúp cho việc tạo container 

Trong mô hình microservice thông thường mỗi service trong đó sẽ có 1 container image riêng 

### 1.1 Quản lý container 

Khi ứng dụng có nhiều container, sẽ xuất hiện các thách thức trong việc quản lý: 
- Làm thê nào để phối hợp hoạt động của nhiều container ?
- Làm thế nào để lập lịch chạy các container ?
- Làm thế nào để tạo các nhóm và phân loại container ? 

Kubernetes là một công nghệ mã nguồn mở giúp quản lý container ở quy mô lớn.

## II. Những khác biệt chính giữa Kubernetes và Docker 

Cả Kubernetes và Docker đều là các công nghệ mã nguồn mở liên quan đến container.

Tuy nhiên, chúng khác nhau về cách hoạt động và vai trò trong việc triển khai ứng dụng.

- Docker được sử dụng để tạo và quản lý container image.
- K8s được sử dụng để quản lý và điều phối nhiều container 

### 2.0 Bảng so sánh 

| Đặc điểm             | Kubernetes                                                           | Docker                                                                       |
| -------------------- | -------------------------------------------------------------------- | ---------------------------------------------------------------------------- |
| **Là gì?**           | Công cụ điều phối container          | Bộ công nghệ container dùng để tạo và chạy container                         |
| **Mục đích sử dụng** | Điều phối nhiều container trên nhiều máy chủ                         | Đóng gói ứng dụng cùng thư viện và môi trường chạy thành container image     |
| **Lợi ích chính**    | Quản lý và vận hành các ứng dụng container hóa phức tạp ở quy mô lớn | Chuẩn hóa môi trường chạy ứng dụng |
