# Phân biệt Docker engine, Containerd và CRI-O

## I. Docker engine là gì ? 
Docker Engine là nền tảng container đầy đủ do Docker phát triển, bao gồm: 
- Docker CLI
- Docker API 
- Docker daemon 
- containerd
- runc 

Khi ta chạy `docker container run nginx` thì luồng hoạt động sẽ là: `docker CLI -> dockerd -> containerd -> runc -> container`

Docker Engine cung cấp các tính năng như: 
- Build image 
- Pull image 
- Push image 
- Network
- Volume 
- Container management 
- Docker Compose 

## II. containerd là gì ? 

`containerd` là container runtime được tách ra từ Docker Engine. Nó chỉ tập trung vào: 
- Pull image
- Run container 
- Stop container 
- Manage container lifecycle

Không có: docker build, docker compose, docker swarm 


### 2.0 Mối quan hệ giữa kubernetes và containerd

K8s cần chạy container, restart container, xóa container chứ không cần Build image hay Docker compose. Do đó k8s sử dụng containerd để quản lý container 

## III. CRI-O là gì ? 

CRI-O là container runtime được thiết kế đặc biệt dành riêng cho kubernetes, cho phép k8s quản lý và chạy các container
