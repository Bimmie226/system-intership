# The big picture

Chúng ta sẽ xem góc nhìn từ 2 phía:

- Góc nhìn Ops
- Góc nhìn Dev

Trong phần góc nhìn Ops, chúng ta sẽ tải một image, khởi chạy một container mới, đăng nhập vào container đó, chạy một lệnh bên trong nó, và sau đó xóa nó.

Trong phần góc nhìn Dev, chúng ta sẽ tập trung nhiều hơn vào ứng dụng. Chúng ta sẽ clone một đoạn mã ứng dụng từ GitHub, kiểm tra một Dockerfile, đóng gói ứng dụng thành container, và chạy nó như một container.

## The Ops Perspective

Khi bạn cài đặt Docker, bạn sẽ có 2 thành phần chính:

- Docker client 
- Docker daemon (Docker engine)

Daemon triển khai runtime, API và mọi thứ cần thiết để chạy Docker

### Images 

Sẽ hữu ích nếu bạn nghĩ Docker image như một đối tượng chứa filesystem của hệ điều hành (OS), một ứng dụng, và tất cả các phụ thuộc của ứng dụng đó. Nếu bạn làm trong mảng vận hành (Ops), nó giống như một template máy ảo (virtual machine template). Một template máy ảo về cơ bản là một máy ảo đã dừng. Trong thế giới Docker, một image thực chất là một container đã dừng.

### Containers

ta có thể dùng lệnh `docker container run` để khởi chạy một container từ nó sau khi đã có image 

### Attaching to running containers

Bạn có thể gắn (attach) shell của mình vào terminal của một container đang chạy bằng lệnh `docker container exec`. 

**Cú pháp:**

```bash
docker container exec <options> <container-name hoặc container-id> <command/app>
```

**Ví dụ:**

```bash
root@labdockersv:~# docker container exec -it distracted_albattani bash
root@ab608aab6eac:/#
```

Trong đó:

- `docker container exec`: Chạy 1 command trong container đang chạy
- `distracted_albattani`: tên container
- `-it`: Cho phép gõ lệnh trực tiếp như đang dùng terminal thật
- `bash`: command được chạy bên trong container 

Thoát khỏi container bằng `Ctrl + P + Q` -> bạn sẽ quay lại terminal của Docker host 

Kiểm tra lại container trên Docker host:

```bash
root@labdockersv:~# docker container ls
CONTAINER ID   IMAGE           COMMAND           CREATED             STATUS             PORTS                                         NAMES
21d5c257ea41   test:latest     "node ./app.js"   47 minutes ago      Up 46 minutes      0.0.0.0:8080->8080/tcp, [::]:8080->8080/tcp   webtest
ab608aab6eac   ubuntu:latest   "/bin/bash"       About an hour ago   Up About an hour                                                 distracted_albattani
root@labdockersv:~#
```

Dừng và xóa container:

```bash
root@labdockersv:~# docker container stop distracted_albattani
distracted_albattani
root@labdockersv:~# docker container rm distracted_albattani
distracted_albattani
root@labdockersv:~#
```

Kiểm tra lại bằng cách liệt kê tất cả container (kể cả đã dừng):

```bash
root@labdockersv:~# docker container ls -a
CONTAINER ID   IMAGE           COMMAND           CREATED             STATUS                           PORTS                                         NAMES
21d5c257ea41   test:latest     "node ./app.js"   49 minutes ago      Up 49 minutes                    0.0.0.0:8080->8080/tcp, [::]:8080->8080/tcp   webtest
bac83338bba6   ubuntu:latest   "/bin/bash"       About an hour ago   Exited (130) About an hour ago                                                 objective_blackwell
root@labdockersv:~#
```

- `-a`: Docker hiển thị cả container đã dừng
