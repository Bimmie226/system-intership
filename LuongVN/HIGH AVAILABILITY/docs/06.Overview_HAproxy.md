# Tìm hiểu HAProxy 

## I. Giới thiệu về HAProxy 

### 1.0 HAProxy là gì ? 
HAProxy là: 

- `TCP Proxy`: có thể cấp nhận các kết nối tcp, kết nối nó tới server cho phép trafic di chuyển theo cả 2 chiều 
- `HTTP reverse-proxy`: Hay còn gọi là gateway, tự bản thân nó có thể là server, nhận các http requests và chuyển các request này tới các server 
- `content-based switch`: dựa vào thành phần của request để xác định server nhận 
- `server load balancer`

### 1.1 Cách thức hoạt động 

Khi HAProxy được bật lên, nó thực hiện 3 điều: 
- Xử lý các incoming connections 
- health check 
- Trao đổi thông tin với các haproxy nodes khác 

### 1.3 Các tính năng cơ bản 
- Proxying 
- SSL
- Monitoring
- HA
- Load balancing 
- ACLs and conditions 
- Content switching 

## II. Chi tiết các tùy chọn cấu hình quan trọng trong HAProxy 

### 2.0 Biến môi trường 
Bên trong HAProxy được bao bọc bởi dấu nháy kép. Nó phải được bắt đầu bằng `$` và nằm bên trong dấu ngoặc nhọn `{}`. Nó có thể chứa chữ cái hoặc dấu `_` nhưng không được phép bắt đầu bằng chữ số 

**Ví dụ:** 

```bash
bind "fd@${FD_APP1}"

log "${LOCAL_SYSLOG}:514" local0 notice 

user "${HAPROXY_USER}"
```

### 2.1 Ví dụ 

```bash
# Simple configuration for an HTTP proxy listening on port 80 on all
# interfaces and forwarding requests to a single backend "servers" with a
# single server "server1" listening on 127.0.0.1:8000
global
    daemon
    maxconn 256

defaults
    mode http
    timeout connect 5000ms
    timeout client 50000ms
    timeout server 50000ms

frontend http-in
    bind *:80
    default_backend servers

backend servers
    server server1 127.0.0.1:8000 maxconn 32


# The same configuration defined with a single listen block. Shorter but
# less expressive, especially in HTTP mode.
global
    daemon
    maxconn 256

defaults
    mode http
    timeout connect 5000ms
    timeout client 50000ms
    timeout server 50000ms

listen http-in
    bind *:80
    server server1 127.0.0.1:8000 maxconn 32
```

Test cấu hình bằng câu lệnh sau: 

```bash
$ sudo haproxy -f <fileconfig.cfg> -c 
```

### 2.2 Global parameters 
Các parameters trong section global thường được sử dụng trong suốt quá trình xử lí và một số cũng là dành riêng cho hệ điều hành 

Một số keyword được hỗ trợ trong section này: 

**Process management and security:** 

- `ca-base`: thư mục mặc định chứa SSL CA certificates and CRLs
- `chroot`: Thay đổi thư mục mặc root của HAProxy 
- `daemon`: chạy background (mode được khuyên dùng khi vận hhành)
- `external-check`: Cho phép user sử dụng external agent để thực hiện health check 
- `log [length] [format] [maxlevel] [minlevel]`: log stdout len 8192 format short local0 debug

**Performance tuning:**

- `max-spread-checks`: Dùng để set thời gian delay giữa lần thực hiện healthcheck cuối cùng và lần check mới 
- `maxconn`: Set số connections tối đa có thể nhận trong cùng 1 thời điểm. Tự động chặn những connection mới khi quá giới hạn 
- `maxconnrate`: Set số connection trên 1 giây 
- `maxcomcpuusage`: Số CPU tối đa mà HAProxy có thể sử dụng 

**Lưu ý:** Đây chỉ là 1 số tùy chọn thông dụng, xem thêm [tại đây](https://www.haproxy.org/download/1.8/doc/configuration.txt)

### 2.3 Proxies 

Các cấu hình proxy có thể được đặt trong 4 section: 
- defaults
- frontend
- backend
- listen

### 2.4 main keywords reference 

- `acl [flags] [operator] ... `: Khai báo acl 
- `balance`: chọn thuật toán LB 
- `bind`: listen IP/port/socket 
- `default_backedn`: khai báo backend được sử dụng khi không có `use_backend` rule nào match 
- `maxconn`: số lượng connection tối đa có thể nhận tại 1 thời điểm 
- `mode {tcp|http|health}`: Mode hoặc protocol cho instance 
- `server`: khai báo server trong backend
