# MariaDB Galera Cluster

MariaDB Galera Cluster là hệ thống cho phép nhiều primary node CSDL MariaDB đồng thời, điều này đồng nghĩa vơi việc ta có thể thực hiện lệnh write trên node bất kỳ và dữ liệu được đồng bộ cho các node khác trong cluster. Nó chỉ hoạt động trên môi trường Linux và InnoDB storage engine 

## I. Giới thiệu về Database Replication 

Replication là quá trình sao chép dữ liêu liên tục từ một database server sang server khác 

Mỗi server được gọi là một node

Mục tiêu: 
- Tất cả node đều có cùng dữ liệu
- Tạo thành một cluster database 
- Ứng dụng client có thể coi cluster như 1 db duy nhất 

### 1.0 Các kiến trúc Replication 

#### 1.0.1 Primary/Replica (Master/Slave) 

Cách hoạt động: 
- Primary: là nguồn dữ liệu chính, chỉ node này được ghi dữ liệu (INSERT, UPDATE, DELETE)
- Replica: nhận dữ liệu từ Primary sau đó thực hiện các truy vấn SQL để đồng bộ dữ liệu 

#### 1.0.2 Multi-Primary Replication 

Trong hệ thống multi-primary mọi node đều là primar, node nào cũng có thể ghi dữ liệu

Khi một node nhận update nó tự động gửi thay đổi sang node khác, tất cả node giữ đồng bộ với nhau

> Đây là mô hình Galera Cluster dùng 

### 1.1 Giao thức Replication 
#### 1.1.1 Asynchronous Replicaiton (đồng bộ bất đồng bộ) 

Primary: 
- commit transaction trước 
- GỬi dữ liệu sang replica sau 

Nhược điểm: 
- Replica có thể bị chậm vài giây - replication lag 
- Nếu primary chết trước khi replica sync xong: có thể mất dữ liệu 

#### 1.1.2 Synchronous Replication (đồng bộ đồng bộ) 

Transaction chỉ được coi là hoàn thành khi tất cả node đều xác nhận thành công 

Khi client nhận OK: dữ liệu tồn tại nhất quán trên toàn cluster

## II. Cấu hình Cluster 

```ini
[mariadb]

bind_address=0.0.0.0
innodb_autoinc_lock_mode=2

wsrep_cluster_name="accounting_cluster"

wsrep_cluster_address="gcomm://192.0.2.1,192.0.2.2,192.0.2.3"

wsrep_provider=/usr/lib/galera/libgalera_enterprise_smm.so
```

### 2.0 Giải thích các biến quan trọng 

**bind_address**

```ini
bind_address=0.0.0.0
```

- Cho phép server nhận TCP/IP từ mọi IP 

**innodb_autoinc_lock_mode**

```ini
innodb_autoinc_lock_mode=2
```

- Galera bắt buộc dùng mode 2

**wsrep_cluster_name**

- Tên cluster, Mọi node phải giống nhau 

**wsrep_cluster_address**

```ini
wsrep_cluster_address="gcomm://192.0.2.1,192.0.2.2,192.0.2.3"
```

- Danh sách node trong cluster 

**wsrep_provider**

- Thư viện Galera replication plugin 

## III. Tính năng của MariaDB Galera Cluster 

- Replication gần như đồng bộ
- Kiến trúc active-active multi-primary 
- Có thể đọc và ghi trên bất kỳ node nào trong cluster 
- Tự động quản lý thành viên: node bị lỗi sẽ bị thoát khỏi cluster
- Tự động join node mới 
- Replication song song thực sự ở mức ROW
- Client kết nối trực tiếp như một MariaDB thông thường

## IV. Một số hạn chế của MariaDB Galera Cluster 
- Chỉ hỗ trợ innoDB 
- các table cần có primary key 
- Tính năng lock/unlock table không hỗ trợ 
- Global transactions chưa hỗ trợ
- Query cache không sử dụng trong Galera Cluster
- Query logs chỉ có thể lưu vào file thay vì vào table
